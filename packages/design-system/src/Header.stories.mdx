The `Header` components provide a collection of semantic HTML heading elements (H1, H2, H3, H4) with consistent typography styling and proper accessibility support. They establish clear content hierarchy while maintaining design system consistency across all heading levels.

Use these components when you need to:

- Create proper document structure with semantic heading hierarchy
- Establish visual content organization with consistent typography
- Build accessible interfaces with screen reader navigation support
- Implement consistent heading styles across your application
- Provide testing-friendly heading elements with data attributes

The component collection supports all standard heading levels (H1-H4), semantic HTML structure, accessibility features, custom styling overrides, and comprehensive testing attribute support for automated testing scenarios.

<br />

The props for all different components are as follows:

- @param id - The id of the header.
- @param data - The object of data attributes that can be used for testing (e.g. data-test or data-cy)
- @param className - The optional className object allows you to override the default styling.
- @param children - The content of the header.
